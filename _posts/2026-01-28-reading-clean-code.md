로버트 C. 마틴의 '클린 코드'를 드디어 다 읽었습니다. 개발자라면 한 번쯤 읽어야 한다는 말을 수없이 들었지만, 이제야 읽게 되었네요.

## 인상 깊었던 내용

### 의미 있는 이름

변수명과 함수명에 의미를 담아야 한다는 점이 가장 와닿았습니다.

```python
# 나쁜 예
d = 7  # elapsed time in days

# 좋은 예
elapsed_time_in_days = 7
```

사소해 보이지만, 이런 습관이 쌓이면 코드의 가독성이 크게 달라집니다.

### 함수는 한 가지만

함수는 하나의 작업만 수행해야 한다는 원칙. 말은 쉽지만 실천은 어렵습니다. 제가 작성한 코드를 돌아보니 하나의 함수에 여러 역할이 뒤섞여 있는 경우가 많았습니다.

### 주석보다 코드로

좋은 코드는 주석이 필요 없다는 말에 처음엔 동의하기 어려웠습니다. 하지만 책을 읽으면서 "주석으로 설명해야 할 정도라면 코드 자체를 개선해야 한다"는 관점에 공감하게 되었습니다.

## 실천하기로 한 것들

1. 변수명과 함수명에 더 시간을 투자하기
2. 함수 길이를 20줄 이내로 유지하기
3. 코드를 작성한 후 반드시 리팩토링 시간 갖기
4. 보이스카우트 규칙 실천하기 - 코드를 발견했을 때보다 깨끗하게 만들기

한 번 읽는 것으로 끝이 아니라, 주기적으로 다시 펼쳐보며 실천하고 있는지 점검해야겠습니다.
